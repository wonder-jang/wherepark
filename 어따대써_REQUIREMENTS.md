# 어따대써 요구사항 명세서

> Android Java 기반 주차 위치 관리 앱 MVP 요구사항 명세서  
> 본 문서는 AI 바이브 코딩 도구(Claude 등)가 구현 방향을 명확히 이해할 수 있도록 기능, 상태, 화면, 알림, DB 구조, 예외 정책을 상세히 정의한다.

---

## 1. 프로젝트 개요

### 1.1 앱 이름

- 앱 이름: **어따대써**
- 앱 목적: 사용자가 차량을 어디에 주차했는지 잊지 않도록, 차량 Bluetooth 연결 상태, 집 Wi-Fi, GPS 위치 정보를 활용해 주차 상황을 감지하고 주차 위치 입력·조회·이력 관리를 제공한다.

### 1.2 핵심 컨셉

앱은 차량 Bluetooth 연결/해제 상태를 중심으로 사용자의 운행/주차 상태를 추정한다.

- 차량 Bluetooth 연결 중: 운행 중으로 판단
- 차량 Bluetooth 연결 해제: 주차 후보로 판단
- 집 Wi-Fi 또는 GPS 집 위치 반경을 통해 집 주차/외부 주차를 분류
- 주차가 감지되면 사용자에게 주차 위치 입력을 요청
- 외부 주차는 Ongoing Notification으로 계속 표시
- 집에서 외출할 때 현재 주차 위치를 알려줌
- 주차 위치 이력, 사진, GPS 좌표를 SQLite에 저장

## 2. 개발 환경

### 2.1 플랫폼

- Android App

### 2.2 개발 언어

- Java

### 2.3 로컬 DB

- SQLite

### 2.4 서버 사용 여부

- MVP에서는 서버를 사용하지 않는다.
- 모든 설정, 상태, 주차 이력, 사진 경로는 기기 로컬에 저장한다.

### 2.5 데이터 저장 위치

- 설정/이력/상태: SQLite
- 사진 파일: 앱 내부 저장소
- SQLite에는 사진 파일 자체를 BLOB으로 저장하지 않고, 사진 파일 경로만 저장한다.

## 4. 상태 정의

앱은 다음 3가지 주요 상태를 관리한다.

### 4.1 주차 상태

내부값 예시:

- `UNKNOWN`
- `DRIVING`
- `PARKED`

화면 표시:

| 내부값 | 표시 |
|---|---|
| `UNKNOWN` | 주차 정보 없음 |
| `DRIVING` | 운행 중 |
| `PARKED` | 주차 중 |

판단 기준:

| 조건 | 상태 |
|---|---|
| 차량 BT 연결됨 | `DRIVING` |
| 차량 BT 연결 해제됨 | `PARKED` 후보 |

### 4.2 집 상태

내부값 예시:

- `UNKNOWN`
- `HOME`
- `AWAY`

화면/의미:

| 내부값 | 의미 |
|---|---|
| `UNKNOWN` | 알 수 없음 |
| `HOME` | 귀가 |
| `AWAY` | 외출 |

판단 기준:

| 조건 | 상태 |
|---|---|
| 저장된 집 Wi-Fi SSID와 현재 연결 Wi-Fi SSID가 일치 | `HOME` |
| 저장된 집 Wi-Fi SSID와 다르거나 Wi-Fi 연결 해제 | `AWAY` |

### 4.3 위치 상태

내부값 예시:

- `UNKNOWN`
- `NEAR_HOME`
- `OUTSIDE`

판단 기준:

| 조건 | 상태 |
|---|---|
| 저장된 집 GPS 위치 기준 설정 반경 이내 | `NEAR_HOME` |
| 저장된 집 GPS 위치 기준 설정 반경 밖 | `OUTSIDE` |

기본 집 위치 반경은 `100m`이며, 사용자가 설정에서 변경할 수 있다.

## 5. 상태 판단 우선순위

### 5.1 확정 우선순위

주차 감지 우선순위는 다음과 같다.

1. 차량 BT 연결 중이면 무조건 `DRIVING`
2. 차량 BT 연결 해제 시 `PARKED` 후보로 판단
3. 주차 판단 시점에 다음 중 하나라도 만족하면 `HOME` 주차로 분류
   - GPS 기준 집 위치 반경 이내
   - 집 Wi-Fi 연결 중
4. 위 조건에 해당하지 않으면 `OUTSIDE` 주차로 분류

### 5.2 차량 BT 우선 원칙

- 차량 BT 연결은 가장 강한 신호로 본다.
- 차량 BT가 연결되면 즉시 운행 상태로 전환한다.
- 차량 BT 연결 시에는 주차 관련 알림을 제거한다.

### 5.3 집/외부 분류 보조 기준

집 주차 판단에는 Wi-Fi와 GPS를 보조 기준으로 사용한다.

- 집 Wi-Fi 또는 GPS 중 하나만 만족해도 집 주차로 기본 판단한다.
- 단, 사용자는 주차 위치 입력 화면에서 `집`/`외부` 구분을 직접 변경할 수 있다.

## 6. 상태 변경 안정화 정책

Bluetooth, Wi-Fi, GPS 상태는 순간적으로 흔들릴 수 있으므로 안정화 지연을 적용한다.

### 6.1 기본 안정화 지연값

| 상태 변경 | 기본값 | 설명 |
|---|---:|---|
| 차량 BT 해제 → 주차 후보 | 30초 | 30초 동안 BT 해제 상태가 유지되면 주차 확정 |
| 차량 BT 연결 → 운행 | 0초 | 즉시 운행 확정 |
| 집 Wi-Fi 연결 → 귀가 후보 | 10초 | 10초 동안 연결 상태 유지 시 귀가 확정 |
| 집 Wi-Fi 해제 → 외출 후보 | 10초 | 10초 동안 해제 상태 유지 시 외출 확정 |
| GPS 집근처 진입 → 집근처 후보 | 30초 | 30초 동안 반경 내 상태 유지 시 집근처 확정 |
| GPS 집근처 이탈 → 외부 후보 | 30초 | 30초 동안 반경 밖 상태 유지 시 외부 확정 |

### 6.2 설정 가능 항목

설정 탭에서 다음 안정화 시간을 사용자가 변경할 수 있다.

- 차량 BT 해제 후 주차 확정 시간
- 집 Wi-Fi 연결 후 귀가 확정 시간
- 집 Wi-Fi 해제 후 외출 확정 시간
- GPS 집근처 진입 확정 시간
- GPS 집근처 이탈 확정 시간

차량 BT 연결 후 운행 확정은 항상 즉시 처리하며, 설정 항목으로 제공하지 않는다.

### 6.3 설정 가능값

| 항목 | 선택 가능값 |
|---|---|
| BT 해제 후 주차 확정 | 0초 / 10초 / 30초 / 60초 / 120초 |
| Wi-Fi 연결/해제 확정 | 0초 / 5초 / 10초 / 30초 / 60초 |
| GPS 진입/이탈 확정 | 0초 / 10초 / 30초 / 60초 / 120초 |

### 6.4 안정화 처리 예시

차량 BT 해제 이벤트 발생 시:

1. `주차 후보` 상태로 기록한다.
2. 설정된 지연 시간 동안 BT 해제 상태가 유지되는지 확인한다.
3. 중간에 차량 BT가 다시 연결되면 주차 후보를 취소한다.
4. 지연 시간이 끝날 때까지 해제 상태가 유지되면 `PARKED`로 확정한다.
5. 주차 위치 입력 요청 알림을 표시한다.

## 7. 첫 실행 및 권한 안내

### 7.1 첫 실행 화면 제공

앱 최초 실행 시 권한과 필수 설정을 안내한다.

### 7.2 앱 소개 예시 문구

> 어따대써  
> 차를 어디에 댔는지 잊지 않도록  
> 블루투스 연결 상태를 기준으로 주차를 감지하고  
> 주차 위치 입력을 도와드립니다.

### 7.3 권한 안내 항목

| 권한/설정 | 사용 목적 |
|---|---|
| Bluetooth | 차량 BT 연결/해제 감지 |
| 위치 | 집 근처 주차 여부 판단, 주차 시점 GPS 좌표 저장 |
| 백그라운드 위치 | 앱이 백그라운드에 있을 때 위치 상태 변화 감지 |
| 알림 | 주차 위치 입력 요청, 주차 위치 상시 알림, 외출 시 주차 위치 알림 |
| Wi-Fi 상태 | 집 Wi-Fi 연결 여부 판단 |
| 배터리 최적화 예외 | 백그라운드 감지 안정화 |

### 7.4 권한 거절 시 정책

- 권한을 거절해도 앱 진입 자체를 막지는 않는다.
- 단, 해당 권한이 필요한 기능은 제한된다.
- 설정 탭의 권한 상태 영역에서 부족한 권한을 안내한다.

예시 문구:

> Bluetooth 권한이 없어 자동 주차 감지를 사용할 수 없습니다.  
> 권한을 허용하면 차량 연결 상태로 주차를 자동 감지할 수 있습니다.

## 8. 설정 탭 요구사항

설정 탭은 다음 섹션으로 구성한다.

1. 필수 설정
2. 집 설정
3. 감지 안정화 설정
4. 권한 상태
5. 백그라운드 감지 안정화 안내
6. 데이터 관리

---

### 8.1 필수 설정

#### 8.1.1 차량 BT 지정

- 차량 BT는 MVP에서 유일한 필수 설정이다.
- 페어링된 Bluetooth 기기 목록에서 선택한다.
- 주변 Bluetooth 스캔 기능은 MVP에서 제외한다.
- 차량 BT가 설정되지 않으면 자동 주차 감지는 사용할 수 없다.

저장 정보:

- `device_name`
- `device_address` 또는 Android에서 접근 가능한 고유 식별값

표시 방식:

- 사용자 화면에는 기기 이름을 표시한다.
- 내부 판단은 가능한 경우 고유 식별값 기준으로 한다.
- Android 버전/권한 제한으로 식별값 사용이 어려우면 이름 기준으로 fallback한다.

전제:

- 차량 BT는 휴대폰과 미리 페어링되어 있어야 한다.

---

### 8.2 집 설정

집 관련 설정은 선택값이다.  
차량 BT만 설정되어도 앱은 기본적으로 외부 주차 기준으로 동작할 수 있다.

#### 8.2.1 집 Wi-Fi 저장

- 사용자가 집 Wi-Fi에 연결된 상태에서 `현재 Wi-Fi를 집 Wi-Fi로 저장` 버튼을 누른다.
- 앱은 현재 연결된 Wi-Fi SSID를 저장한다.
- 주변 Wi-Fi 목록을 보여주는 기능은 MVP에서 제외한다.

동작:

- 저장된 SSID와 현재 연결 Wi-Fi SSID가 일치하면 집 상태를 `HOME`으로 판단한다.
- 다르거나 Wi-Fi 연결이 끊기면 `AWAY`로 판단한다.

권한/제약:

- Android 버전에 따라 위치 권한 또는 Wi-Fi 상태 접근 권한이 필요할 수 있다.
- SSID를 가져올 수 없으면 저장 실패 안내를 표시한다.

#### 8.2.2 집 위치 저장

- 사용자가 집에서 `현재 위치를 집으로 저장` 버튼을 누른다.
- 앱은 현재 GPS 좌표를 집 위치로 저장한다.
- 지도에서 직접 위치를 찍는 기능은 MVP에서 제외한다.

#### 8.2.3 집 위치 반경

사용자가 집근처 판단 반경을 선택할 수 있다.

선택값:

- `50m`
- `100m`
- `200m`
- `300m`

기본값:

- `100m`

동작:

- 집 위치가 설정된 경우에만 반경 설정이 GPS 기반 집/외부 판단에 사용된다.
- 집 위치가 설정되지 않은 경우 반경 설정값은 판단에 사용하지 않는다.

#### 8.2.4 집 주차장 층수 설정

집 주차장은 지하와 지상을 모두 지원한다.

설정 항목:

- 집 주차장 지하 층수
- 집 주차장 지상 층수

기본값:

| 항목 | 기본값 |
|---|---:|
| 지하 층수 | 1 |
| 지상 층수 | 1 |

제약:

- 지하 층수 최소값: `1`
- 지상 층수 최소값: `1`
- `0층` 설정은 허용하지 않는다.

예시:

- 지하 층수 3, 지상 층수 2인 경우:
  - 지하: `B1F`, `B2F`, `B3F`
  - 지상: `1F`, `2F`
  - 기타

### 8.3 감지 안정화 설정

설정 탭에서 상태 변경 안정화 시간을 변경할 수 있다.

표시 항목:

- 집 Wi-Fi 연결 후 귀가 확정 시간
- 집 Wi-Fi 해제 후 외출 확정 시간
- GPS 집근처 진입 확정 시간

기본값:

| 항목 | 기본값 |
|---|---:|
| 차량 BT 해제 후 주차 확정 | 30초 |
| 집 Wi-Fi 연결 후 귀가 확정 | 10초 |
| 집 Wi-Fi 해제 후 외출 확정 | 10초 |
| GPS 집근처 진입 확정 | 30초 |
| GPS 집근처 이탈 확정 | 30초 |

---

### 8.4 권한 상태 영역

설정 탭에 권한 상태 영역을 제공한다.

표시 항목:

- Bluetooth 권한
- 위치 권한
- 백그라운드 위치 권한
- 알림 권한
- Wi-Fi 상태 접근 가능 여부
- 배터리 최적화 상태

상태값:

| 상태 | 의미 |
|---|---|
| `허용됨` | 정상 사용 가능 |
| `필요` | 권한 요청 또는 설정 필요 |
| `확인 필요` | Android 버전/기기 정책상 직접 확인 필요 |

동작:

- 설정 탭 진입 시 최신 상태로 갱신한다.
- 권한이 부족하면 재요청 또는 Android 설정 화면 이동 버튼을 제공한다.

---

### 8.5 백그라운드 감지 안정화 안내

설정 탭에 `백그라운드 감지 안정화` 안내 영역을 제공한다.

목적:

- Android 배터리 최적화 또는 제조사별 백그라운드 제한으로 자동 감지가 누락될 수 있음을 사용자에게 안내한다.

예시 문구:

> 백그라운드 감지 안정화  
> 배터리 최적화가 켜져 있으면 차량 블루투스 연결 해제나 Wi-Fi 변경을 놓칠 수 있습니다.  
> 자동 감지를 안정적으로 사용하려면 배터리 최적화 예외 설정을 확인해주세요.

제공 기능:

- 배터리 최적화 예외 설정 안내
- 가능한 경우 Android 설정 화면 이동 버튼
- 권한 또는 설정 부족 상태 표시

상태 예시:

- `정상`
- `권한 필요`
- `배터리 최적화 확인 필요`

### 8.6 데이터 관리

설정 탭 하단에 데이터 관리 섹션을 제공한다.

제공 기능:

| 기능 | 설명 |
|---|---|
| 모든 주차 이력 삭제 | SQLite의 모든 주차 이력 삭제 |
| 모든 사진 파일 삭제 | 앱 내부 저장소의 주차 사진 파일 전부 삭제 |
| 앱 설정 초기화 | 차량 BT, 집 Wi-Fi, 집 위치, 안정화 시간 등 초기화 |
| 전체 데이터 초기화 | 이력, 사진, 설정, 현재 상태 모두 초기화 |

공통 정책:

- 모든 삭제/초기화 기능 실행 전 확인 다이얼로그를 표시한다.
- 삭제 작업은 복구하지 않는다.

예시 문구:

> 이 작업은 되돌릴 수 없습니다.

#### 8.6.1 모든 주차 이력 삭제

- 주차 이력 데이터를 모두 삭제한다.
- 현재 주차 정보도 `주차 정보 없음`으로 변경한다.
- Ongoing Notification을 제거한다.
- 사진 파일은 별도 삭제 기능을 통해 삭제한다.

#### 8.6.2 모든 사진 파일 삭제

- 앱 내부 저장소에 저장된 주차 사진 파일을 모두 삭제한다.
- SQLite의 사진 경로 값을 비운다.
- 주차 이력 자체는 삭제하지 않는다.

#### 8.6.3 앱 설정 초기화

초기화 대상:

- 차량 BT 설정
- 집 Wi-Fi 설정
- 집 위치 설정
- 집 위치 반경
- 집 주차장 지하/지상 층수
- 감지 안정화 시간 설정
- 첫 실행 완료 여부는 구현 정책에 따라 초기화 여부 결정 가능

초기화 후:

- 차량 BT 미설정 상태가 된다.
- 주차 위치 탭에 차량 BT 설정 안내를 표시한다.
- 기본값이 있는 설정은 기본값으로 복원한다.

#### 8.6.4 전체 데이터 초기화

- 모든 주차 이력 삭제
- 모든 사진 파일 삭제
- 앱 설정 초기화
- 현재 주차 정보 초기화
- Ongoing Notification 제거

## 9. 주차 위치 탭 요구사항

### 9.1 기본 역할

주차 위치 탭은 현재 주차 상태와 현재 주차 위치를 보여주는 메인 화면이다.

### 9.2 표시 상태

| 상태 | 화면 표시 | 버튼 |
|---|---|---|
| 차량 BT 미설정 | 차량 BT 설정 필요 안내 | 설정 탭 이동 |
| 정보 없음 | 주차 정보 없음 | 수동 저장 가능 |
| 운행 중 | 운행 중 | 없음 또는 수동 저장 |
| 주차 + 위치 미입력 | 주차 위치 미입력 | 주차 위치 입력 / 초기화 |
| 주차 + 위치 입력됨 | 저장된 주차 위치 표시 | 수정 / 초기화 / 지도에서 보기 조건부 |

### 9.3 차량 BT 미설정 안내

차량 BT가 설정되지 않은 경우:

> 차량 BT 설정이 필요합니다  
> 설정 탭에서 차량 블루투스를 지정해주세요.

### 9.4 현재 주차 위치 표시

표시 예시:

- `집 · 지하 B2F · 103동 엘리베이터 앞`
- `집 · 지상 1F · 관리사무소 앞`
- `외부 · 지하 B1F · A구역 기둥 12번`
- `외부 · 지상 3F · 엘리베이터 앞`
- `외부 · 기타 · 운동장 옆 임시주차장`

### 9.5 직접 입력/수정

주차 위치 탭에서 직접 주차 위치를 입력하거나 수정할 수 있다.

- `주차 위치 입력` 버튼
- `수정` 버튼
- `수동 저장` 버튼
- `초기화` 또는 `현재 주차 정보 삭제` 버튼

동작:

- 입력/수정/수동 저장은 모두 주차 위치 입력 화면으로 이동한다.
- 수정 완료 시 기존 주차 정보를 덮어쓴다.
- 외부 주차 Ongoing Notification이 있는 경우 내용도 갱신한다.

### 9.6 수동 저장

자동 감지가 실패하거나 사용자가 직접 주차 위치를 기록하고 싶은 경우 수동 저장을 지원한다.

동작:

- 주차 위치 탭에 `수동 저장` 버튼을 제공한다.
- 사용자가 수동 저장을 누르면 주차 위치 입력 화면으로 이동한다.
- 수동 저장 시 주차 상태를 `PARKED`로 변경한다.
- 수동 저장한 기록도 주차 이력에 저장한다.
- 저장 방식은 `MANUAL`로 저장한다.
- 외부 주차로 저장된 경우 Ongoing Notification을 등록한다.

수동 저장 시 집/외부 기본 판단:

- 집 Wi-Fi 연결 중이거나 GPS 기준 집 반경 이내이면 기본값 `집`
- 그 외에는 기본값 `외부`
- 집 관련 설정이 없으면 기본값 `외부`

### 9.7 현재 주차 정보 초기화

사용자가 현재 주차 정보를 초기화할 수 있다.

확인 문구 예시:

> 현재 주차 정보를 삭제할까요?

확인 시:

- 현재 주차 위치 표시를 `주차 정보 없음`으로 변경
- Ongoing Notification 제거
- 주차 상태는 `UNKNOWN` 또는 정보 없음 상태로 변경
- 연결된 주차 이력은 삭제하지 않음
- 해당 이력의 `현재 주차` 표시만 해제

## 10. 주차 상태 변경 흐름

### 10.1 주차 상태가 `PARKED`로 변경되는 경우

조건:

- 차량 BT 연결 해제 상태가 설정된 안정화 시간 동안 유지됨
- 또는 집 상태 귀가 이벤트가 발생했는데 주차 상태가 아직 `PARKED`가 아님
- 또는 사용자가 수동 저장 실행

동작:

1. 주차 상태를 `PARKED`로 변경한다.
2. GPS 좌표 저장을 시도한다.
3. 집/외부 기본 구분을 판단한다.
4. 주차 위치 입력 요청 알림을 표시한다.
5. 주차 위치가 입력되지 않은 상태이면 `주차 위치 미입력`으로 표시한다.

### 10.2 주차 위치 입력을 하지 않은 경우

사용자가 알림을 무시하거나 입력 화면을 취소한 경우:

- 주차 상태는 `PARKED`로 유지
- 주차 위치는 `미입력` 상태로 저장 또는 표시
- 주차 위치 탭에는 `주차 위치 미입력` 표시
- `주차 위치를 입력하세요` 알림은 계속 유지
- 사용자가 알림을 누르면 언제든 다시 입력 가능

### 10.3 주차 상태가 `DRIVING`으로 변경되는 경우

조건:

- 차량 BT 연결 감지

동작:

- 즉시 `DRIVING`으로 전환
- 주차 위치 입력 요청 알림 제거
- 외부 주차 위치 Ongoing Notification 제거
- 주차 위치 탭에 `운행 중` 표시
- 주차 이력은 삭제하지 않음
- 현재 주차 여부는 구현 정책상 운행 전환 시 해제하거나, 다음 주차 감지 시 갱신할 수 있다. MVP에서는 운행 전환 시 현재 주차 알림은 제거하되 이력은 유지한다.

## 11. 집 상태 변경 흐름

### 11.1 집 상태가 `HOME`으로 변경되는 경우

조건:

- 집 Wi-Fi 연결 상태가 안정화 시간 동안 유지됨

동작:

- 현재 주차 상태가 이미 `PARKED`라면 아무것도 하지 않는다.
- 현재 주차 상태가 `PARKED`가 아니라면:
  1. 주차 상태를 `PARKED`로 변경한다.
  2. 주차 위치 입력 요청 흐름을 실행한다.

목적:

- 차량 BT 감지가 실패했더라도 집 Wi-Fi 연결로 귀가/주차를 보완 감지한다.

### 11.2 집 상태가 `AWAY`로 변경되는 경우

조건:

- 집 Wi-Fi 해제 상태가 안정화 시간 동안 유지됨
- 현재 주차 상태가 `PARKED`

동작:

- 외출 시 주차 위치 알림을 표시한다.
- 진동을 발생시킨다.
- 현재 저장된 주차 위치를 알려준다.

목적:

- 사용자가 집을 나설 때 차 위치를 바로 확인할 수 있도록 한다.

## 12. 주차 위치 입력 화면 요구사항

### 12.1 화면 방식

주차 위치 입력은 다이얼로그가 아니라 **입력 전용 화면**으로 구성한다.

### 12.2 진입 경로

다음 동작은 모두 동일한 입력 화면으로 이동한다.

- 주차 위치 입력 요청 알림 클릭
- 주차 위치 탭의 `주차 위치 입력` 버튼 클릭
- 주차 위치 탭의 `수정` 버튼 클릭
- 주차 위치 탭의 `수동 저장` 버튼 클릭
- 주차 이력 상세의 `수정` 버튼 클릭

### 12.3 입력 항목

입력 화면 제공 항목:

- 주차 구분: `집` / `외부`
- 주차 유형: `지하` / `지상` / `기타`
- 층 선택
- 추가 위치 정보 텍스트
- 사진 촬영
- 갤러리 사진 선택
- 사진 미리보기
- 사진 삭제
- 저장 버튼
- 취소 버튼

### 12.4 주차 구분 변경

앱이 자동 판단한 집/외부 구분을 기본값으로 사용하되, 사용자가 직접 변경할 수 있다.

| 선택 구분 | 입력 UI |
|---|---|
| 집 | 집 주차장 설정 기준의 지하/지상/기타 선택 |
| 외부 | 외부 기준의 지하/지상/기타 선택 |

목적:

- 집 근처 상가에 주차했는데 집으로 판단되는 경우 보정
- GPS가 튀어 외부로 판단됐지만 실제로는 집인 경우 보정

### 12.5 집 주차 입력 UI

집 주차는 다음 유형을 지원한다.

| 주차 유형 | 선택값 |
|---|---|
| 지하 | 설정한 지하 층수만큼 `B1F ~ BnF` |
| 지상 | 설정한 지상 층수만큼 `1F ~ nF` |
| 기타 | 층 선택 없음 |

기본값:

- 주차 유형: `지하`
- 층: `B1F`

예시:

- 집 지하 `B2F` 103동 엘리베이터 앞
- 집 지상 `1F` 관리사무소 앞
- 집 기타 단지 외부 노상

### 12.6 외부 주차 입력 UI

외부 주차는 다음 유형을 지원한다.

| 주차 유형 | 선택값 |
|---|---|
| 지하 | `B1F ~ B100F` |
| 지상 | `1F ~ 100F` |
| 기타 | 층 선택 없음 |

기본값:

- 주차 유형: `지하`
- 층: `B1F`

예시:

- 외부 지하 `B1F` A구역 기둥 12번
- 외부 지상 `3F` 엘리베이터 앞
- 외부 기타 운동장 옆 임시주차장

### 12.7 입력 필수값

의미 없는 빈 기록이 저장되지 않도록 다음 필수값 정책을 적용한다.

| 상황 | 필수값 |
|---|---|
| 집 지하 | 층 선택 필수 |
| 집 지상 | 층 선택 필수 |
| 집 기타 | 추가 위치 정보 또는 사진 중 하나 이상 필수 |
| 외부 지하 | 층 선택 필수 |
| 외부 지상 | 층 선택 필수 |
| 외부 기타 | 추가 위치 정보 또는 사진 중 하나 이상 필수 |

선택값:

- 추가 위치 정보
- 사진
- GPS 좌표

단, `기타` 유형에서는 층 정보가 없으므로 추가 위치 정보 또는 사진 중 하나 이상이 필요하다.

### 12.8 저장 동작

저장 시:

1. 입력값 검증
2. 사진이 있으면 압축 저장
3. GPS 좌표가 있으면 함께 저장
4. 주차 이력 생성 또는 수정
5. 현재 주차 정보로 지정 또는 갱신
6. 알림 갱신
7. 이전 화면으로 복귀

신규 저장인 경우:

- 주차 이력 생성
- 현재 주차 정보로 지정
- 외부 주차이면 Ongoing Notification 등록
- 집 주차이면 Ongoing Notification은 등록하지 않음

수정인 경우:

- 해당 주차 이력 업데이트
- 현재 주차 이력을 수정한 경우 주차 위치 탭 표시 갱신
- Ongoing Notification이 있으면 내용 갱신

### 12.9 취소 동작

신규 주차 입력 중 취소:

- 주차 상태는 `PARKED` 유지
- 주차 위치는 `미입력` 상태 유지
- 주차 위치 입력 요청 알림 유지

기존 주차 정보 수정 중 취소:

- 기존 값 변경 없음

## 13. 주차 이력 탭 요구사항

### 13.1 기본 역할

주차 이력 탭은 과거 주차 기록을 최신순으로 보여준다.

### 13.2 하단 탭 표시

하단 탭 중 두 번째 탭으로 제공한다.

### 13.3 이력 저장 정책

- 주차가 감지되거나 수동 저장될 때마다 SQLite에 주차 이력을 저장한다.
- 현재 주차 위치는 가장 최근 또는 현재 주차로 지정된 이력을 기준으로 표시한다.
- 운행 상태로 변경되어도 이력은 삭제하지 않는다.
- 이력은 사용자가 직접 삭제하지 않는 한 무제한 보관한다.

### 13.4 리스트 표시

주차 이력은 최신순으로 표시한다.

각 이력 항목 최소 표시 정보:

- 주차 일시
- 주차 구분: `집` / `외부`
- 주차 유형: `지하` / `지상` / `기타`
- 층 정보
- 추가 위치 정보
- 사진 있음 아이콘
- 현재 주차 배지 조건부

예시:

> 현재 주차  
> 외부 · 지하 B2F · A구역 기둥 12번 📷  
> 2026-06-24 21:10

> 집 · 지상 1F · 관리사무소 앞  
> 2026-06-22 19:30

> 외부 · 기타 · 운동장 옆 임시주차장  
> 2026-06-19 18:10

### 13.5 현재 주차 배지

현재 주차 정보가 있는 경우, 해당 주차 이력에 `현재 주차` 배지를 표시한다.

표시 위치:

- 주차 이력 리스트 항목
- 주차 이력 상세 화면/상세 다이얼로그

초기화/삭제 시:

- 현재 주차 정보를 초기화하면 해당 이력의 `현재 주차` 배지를 제거한다.
- 현재 주차 이력을 삭제하면 현재 주차 정보도 초기화한다.

### 13.6 검색 기능

주차 이력 탭 상단에 검색창을 제공한다.

검색 대상:

- 주차 구분: `집`, `외부`
- 주차 유형: `지하`, `지상`, `기타`
- 층 정보: `B2F`, `3F` 등
- 추가 위치 정보
- 주차 일시 텍스트

검색 방식:

- 단순 포함 검색
- 대소문자 구분 없음
- 검색어가 비어 있으면 전체 이력 표시

### 13.7 필터 기능

검색창과 함께 필터 칩을 제공한다.

필터 항목:

- `전체`
- `집`
- `외부`

동작:

- 기본값은 `전체`
- 정렬은 최신순 고정
- 검색어와 필터는 함께 적용한다.

예시:

- 필터: `외부`
- 검색어: `스타필드`
- 결과: 외부 주차 이력 중 `스타필드`가 포함된 이력만 표시

### 13.8 사진 표시 방식

리스트에서는 사진 원본이나 썸네일을 직접 보여주지 않는다.

- 사진이 있으면 사진 아이콘만 표시한다.
- 실제 사진 확인은 상세 화면/상세 다이얼로그에서 제공한다.

### 13.9 이력 상세

이력 항목을 누르면 상세 화면 또는 상세 다이얼로그를 제공한다.

표시 정보:

- 주차 일시
- 주차 구분
- 주차 유형
- 층 정보
- 추가 위치 정보
- 현재 주차 여부
- 저장 방식: 자동 감지 / 수동 저장
- GPS 좌표 저장 여부
- 사진
- 지도에서 보기 버튼 조건부

제공 기능:

- 수정
- 삭제
- 지도에서 보기 조건부

### 13.10 저장 방식 표시

저장 방식은 주차 이력 상세에서만 표시한다.

| 내부값 | 화면 표시 |
|---|---|
| `AUTO` | 자동 감지 |
| `MANUAL` | 수동 저장 |

리스트와 주차 위치 탭에는 저장 방식을 표시하지 않는다.

### 13.11 이력 수정

- 상세 화면에서 `수정` 버튼을 누르면 입력 화면으로 이동한다.
- 기존 값이 채워진 상태로 표시한다.
- 수정 완료 시 해당 이력을 업데이트한다.
- 현재 주차 이력을 수정한 경우 주차 위치 탭과 알림도 함께 갱신한다.

### 13.12 이력 삭제

- 상세 화면에서 `삭제` 버튼 제공
- 삭제 전 확인 다이얼로그 표시

예시:

> 이 주차 이력을 삭제할까요?

삭제 정책:

- 삭제한 이력은 복구하지 않는다.
- 연결된 사진 파일도 함께 삭제한다.
- 현재 주차 이력을 삭제한 경우:
  - 현재 주차 정보 초기화
  - 주차 위치 탭 `주차 정보 없음`
  - Ongoing Notification 제거

## 14. 알림 요구사항

알림은 총 3종으로 구분한다.

| 알림 | 유형 | 유지 여부 |
|---|---|---|
| 주차 위치 입력 요청 알림 | 입력 유도 | 입력 완료 전까지 유지 |
| 외부 주차 위치 상시 알림 | Ongoing | 운행 전환 또는 현재 주차 초기화 전까지 유지 |
| 외출 시 주차 위치 알림 | 일회성 | 자동 사라짐 가능 |

### 14.1 주차 위치 입력 요청 알림

표시 조건:

- 주차가 감지됐지만 주차 위치가 입력되지 않은 경우

문구 예시:

> 주차 위치를 입력하세요

동작:

- 알림을 누르면 주차 위치 입력 화면으로 이동한다.
- 입력 완료 전까지 알림을 유지한다.
- 입력 완료 시 제거한다.
- 외부 주차 위치가 입력된 경우 외부 주차 위치 상시 알림으로 전환한다.

### 14.2 외부 주차 위치 상시 알림

표시 조건:

- 외부 주차 위치가 입력된 경우

문구 예시:

> 주차 위치: 외부 지하 B2F A구역 기둥 12번

동작:

- Ongoing Notification으로 표시한다.
- 알림을 누르면 주차 위치 탭으로 이동한다.
- 현재 주차 위치를 수정하면 알림 내용도 갱신한다.
- 다음 상황에서 제거한다.
  - 주차 상태가 `DRIVING`으로 변경
  - 현재 주차 정보 초기화
  - 현재 주차 이력 삭제

### 14.3 외출 시 주차 위치 알림

표시 조건:

- 집 상태가 `AWAY`로 변경됨
- 현재 주차 상태가 `PARKED`
- 현재 주차 정보가 존재함

문구 예시:

> 차 위치를 확인하세요  
> 집 지하 B2F 103동 엘리베이터 앞

동작:

- 진동 발생
- 알림을 누르면 주차 위치 탭으로 이동
- 일회성 알림으로 처리
- 자동으로 사라질 수 있음

## 15. 백그라운드 감지 요구사항

### 15.1 기본 방향

어따대써는 앱을 열지 않아도 상태 변화를 감지할 수 있도록 백그라운드 감지를 지원한다.

### 15.2 감지 대상

- 차량 BT 연결/해제
- 집 Wi-Fi 연결/해제
- 집 위치 반경 진입/이탈

### 15.3 동작

- 앱이 백그라운드에 있어도 차량 BT 해제를 감지하면 주차 후보로 판단한다.
- 안정화 지연 후 주차 확정 시 주차 위치 입력 요청 알림을 표시한다.
- 차량 BT 연결 시 즉시 운행 상태로 전환하고 관련 알림을 제거한다.
- 집 Wi-Fi 연결/해제, GPS 위치 상태 변화도 보조 판단에 사용한다.

### 15.4 구현 고려 요소

Android 버전별 백그라운드 제한을 고려해 다음 요소를 검토한다.

- BroadcastReceiver
- Foreground Service
- Notification Channel
- 권한 허용 상태 확인
- 배터리 최적화 예외 안내
- 위치 권한 및 백그라운드 위치 권한 안내

### 15.5 사용자 안내

백그라운드 감지가 Android 버전, 제조사 정책, 배터리 최적화에 따라 제한될 수 있음을 설정 탭에서 안내한다.

## 16. GPS 좌표 저장 및 지도 연동

### 16.1 GPS 좌표 저장

주차가 감지되거나 수동 저장을 수행할 때 가능한 경우 현재 GPS 좌표를 함께 저장한다.

저장 정보:

- GPS 위도
- GPS 경도
- GPS 저장 성공 여부

정책:

- 위치 권한이 있고 현재 위치를 가져올 수 있으면 저장한다.
- 위치 권한이 없거나 위치를 가져오지 못하면 좌표 없이 저장한다.
- GPS 저장 실패가 주차 위치 저장을 막지는 않는다.

개인정보 안내:

> 주차 위치 확인과 이력 관리를 위해 주차 시점의 위치 좌표를 기기에 저장할 수 있습니다. 저장된 정보는 앱 내부 SQLite DB에 보관됩니다.

### 16.2 지도에서 보기

GPS 좌표가 저장된 주차 기록에는 `지도에서 보기` 버튼을 제공한다.

제공 위치:

- 주차 위치 탭
- 주차 이력 상세 화면/상세 다이얼로그

표시 조건:

- 해당 기록에 GPS 위도/경도 값이 있음

동작:

- 외부 지도 앱을 실행한다.
- Google Maps, 카카오맵, 네이버지도 등 사용 가능
- MVP에서는 앱 내부 지도 SDK를 사용하지 않는다.
- 지도 앱이 없는 경우 브라우저 지도 URL fallback을 고려한다.

## 17. 사진 저장 요구사항

### 17.1 사진 저장 기능

주차 위치 입력 시 사진 1장을 함께 저장할 수 있다.

목적:

- 기둥 번호
- 주차 구역 표지판
- 엘리베이터 위치
- 주변 표식
- 출입구 번호

### 17.2 등록 방식

다음 두 가지를 모두 지원한다.

1. 사진 촬영
2. 갤러리에서 사진 선택

### 17.3 제한

- 주차 기록 1개당 사진은 최대 1장
- 사진은 선택 입력
- 사진 없이도 주차 위치 저장 가능
- 단, `기타` 유형에서는 추가 위치 정보가 없을 경우 사진이 필수 역할을 할 수 있음

### 17.4 저장 방식

- 원본 사진은 저장하지 않는다.
- 앱용 압축본만 앱 내부 저장소에 저장한다.
- SQLite에는 압축본 파일 경로를 저장한다.
- SQLite에 이미지 BLOB 저장 금지

권장 압축 기준:

- 긴 변 기준 최대 `1280px`
- JPEG 압축 품질 `80%`
- 파일 형식: `JPEG`

### 17.5 사진 수정/삭제

- 기존 사진이 있으면 미리보기 표시
- 새 사진을 선택하거나 촬영하면 기존 사진을 교체
- 기존 사진 교체 시 이전 압축본 파일 삭제
- 사진 삭제 시 해당 주차 기록의 사진 경로 제거 및 파일 삭제
- 주차 이력 삭제 시 연결된 사진 파일도 함께 삭제

### 17.6 사진 표시

- 주차 위치 탭: 현재 주차 사진이 있으면 표시 가능
- 주차 이력 리스트: 사진 아이콘만 표시
- 주차 이력 상세: 실제 사진 표시

- 계정 기반 데이터 백업

## 19. SQLite DB 설계 초안

> 실제 구현 시 Android Java의 SQLiteOpenHelper 또는 Room 미사용 순수 SQLite 구현 중 선택 가능하다. MVP 명세에서는 테이블 구조와 주요 컬럼을 정의한다.

---

### 19.1 `app_settings`

앱 설정값 저장 테이블.

MVP에서는 key-value 구조 또는 단일 row 구조 중 하나를 선택할 수 있다. 구현 단순성을 위해 단일 row 구조를 권장한다.

#### 주요 컬럼

| 컬럼 | 타입 예시 | 설명 |
|---|---|---|
| `id` | INTEGER PRIMARY KEY | 설정 row ID. 단일 row 사용 시 1 고정 가능 |
| `vehicle_bt_name` | TEXT | 차량 BT 표시 이름 |
| `vehicle_bt_address` | TEXT | 차량 BT 식별값 또는 주소 |
| `home_wifi_ssid` | TEXT | 집 Wi-Fi SSID |
| `home_latitude` | REAL | 집 GPS 위도 |
| `home_longitude` | REAL | 집 GPS 경도 |
| `home_radius_meters` | INTEGER | 집 위치 반경. 기본 100 |
| `home_underground_floor_count` | INTEGER | 집 지하 층수. 기본 1, 최소 1 |
| `home_ground_floor_count` | INTEGER | 집 지상 층수. 기본 1, 최소 1 |
| `bt_disconnect_stabilize_seconds` | INTEGER | BT 해제 후 주차 확정 시간. 기본 30 |
| `wifi_connect_stabilize_seconds` | INTEGER | Wi-Fi 연결 후 귀가 확정 시간. 기본 10 |
| `wifi_disconnect_stabilize_seconds` | INTEGER | Wi-Fi 해제 후 외출 확정 시간. 기본 10 |
| `gps_enter_stabilize_seconds` | INTEGER | GPS 집근처 진입 확정 시간. 기본 30 |
| `gps_exit_stabilize_seconds` | INTEGER | GPS 집근처 이탈 확정 시간. 기본 30 |
| `is_onboarding_completed` | INTEGER | 첫 실행 완료 여부. 0/1 |
| `created_at` | TEXT | 생성 일시 |
| `updated_at` | TEXT | 수정 일시 |

---

### 19.2 `parking_records`

주차 이력 저장 테이블.

#### 주요 컬럼

| 컬럼 | 타입 예시 | 설명 |
|---|---|---|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | 주차 기록 ID |
| `parked_at` | TEXT | 주차 감지/수동 저장 일시 |
| `is_current` | INTEGER | 현재 주차 여부. 0/1 |
| `save_type` | TEXT | `AUTO` / `MANUAL` |
| `parking_place_type` | TEXT | `HOME` / `OUTSIDE` |
| `parking_level_type` | TEXT | `UNDERGROUND` / `GROUND` / `ETC` |
| `floor_label` | TEXT | `B2F`, `3F` 등. 기타는 NULL 가능 |
| `memo` | TEXT | 추가 위치 정보 |
| `latitude` | REAL | 주차 시점 GPS 위도 |
| `longitude` | REAL | 주차 시점 GPS 경도 |
| `has_gps` | INTEGER | GPS 좌표 저장 여부. 0/1 |
| `photo_path` | TEXT | 압축 사진 파일 경로 |
| `created_at` | TEXT | 생성 일시 |
| `updated_at` | TEXT | 수정 일시 |

#### 값 정의

`save_type`:

- `AUTO`: 자동 감지
- `MANUAL`: 수동 저장

`parking_place_type`:

- `HOME`: 집
- `OUTSIDE`: 외부

`parking_level_type`:

- `UNDERGROUND`: 지하
- `GROUND`: 지상
- `ETC`: 기타

#### 제약/정책

- `is_current = 1`인 record는 최대 1개만 유지한다.
- 새로운 현재 주차 기록을 저장할 때 기존 현재 주차 기록의 `is_current`는 0으로 변경한다.
- 현재 주차 초기화 시 이력은 삭제하지 않고 `is_current`만 0으로 변경한다.
- 이력 삭제 시 연결된 사진 파일도 삭제한다.

## 21. 구현 시 주의사항

### 21.1 Android 권한 및 버전 대응

Android 버전에 따라 Bluetooth, Wi-Fi SSID, 백그라운드 위치, 알림 권한 요구사항이 다를 수 있다.

구현 시 다음을 반드시 고려한다.

- Android 12 이상 Bluetooth 권한
- Android 13 이상 알림 권한
- 위치 권한 및 백그라운드 위치 권한
- Wi-Fi SSID 접근 제약
- 배터리 최적화 정책
- 제조사별 백그라운드 제한

### 21.2 자동 감지 신뢰성

자동 감지는 100% 정확하지 않을 수 있다.

따라서 다음 보정 기능을 반드시 제공한다.

- 수동 저장
- 현재 주차 정보 초기화
- 입력 화면에서 집/외부 구분 변경
- 주차 위치 수정
- 권한 상태 표시
- 상태 로그 저장

### 21.3 알림 UX

- 주차 위치 입력 요청 알림은 입력 완료 전까지 유지한다.
- 외부 주차 위치 알림은 Ongoing Notification으로 유지한다.
- 집 주차는 Ongoing Notification을 등록하지 않는다.
- 운행 상태 전환 또는 현재 주차 정보 초기화 시 관련 알림을 제거한다.

### 21.4 사진 파일 관리

- 사진은 원본 저장 금지
- 압축본만 저장
- 교체/삭제/이력 삭제 시 파일 누수 없도록 처리
- SQLite에는 파일 경로만 저장

### 21.5 현재 주차 정보와 이력 관계

- 현재 주차 정보는 `parking_records.is_current = 1`인 기록을 기준으로 한다.
- 현재 주차 정보 초기화는 이력 삭제가 아니라 `is_current = 0` 처리다.
- 현재 이력을 삭제하면 현재 주차 정보도 초기화한다.

### 21.6 Claude 구현 지시용 핵심 요약

Claude가 구현할 때 반드시 지켜야 할 핵심 원칙:

1. Android Java + SQLite 로컬 앱으로 구현한다.
2. 하단 탭은 주차 위치 / 주차 이력 / 설정 3개다.
3. 차량 BT가 운행/주차 판단의 최우선 신호다.
4. 차량 BT 연결은 즉시 운행, 차량 BT 해제는 안정화 지연 후 주차다.
5. 집/외부 판단은 집 Wi-Fi 또는 GPS 반경으로 한다.
6. 사용자는 입력 화면에서 집/외부 구분을 수정할 수 있다.
7. 주차 입력은 다이얼로그가 아니라 전용 화면이다.
8. 집/외부 모두 지하, 지상, 기타를 지원한다.
9. 층 표기는 `B1F`, `B2F`, `1F`, `2F` 형식이다.
10. 사진은 1장만 저장하며 압축본 파일 경로만 DB에 저장한다.
11. GPS 좌표는 가능하면 주차 기록에 함께 저장한다.
12. 외부 주차는 Ongoing Notification으로 표시한다.
13. 집에서 외출할 때 현재 주차 위치를 진동 알림으로 알려준다.
14. 주차 이력은 무제한 보관하고, 상태 로그는 최근 1,000개만 보관한다.
15. MVP에서는 백업/복원/서버/계정/내부 지도 SDK를 제외한다.

## 22. 용어 정리

| 용어 | 의미 |
|---|---|
| 차량 BT | 사용자가 설정한 차량 Bluetooth 기기 |
| 주차 상태 | 운행/주차/알 수 없음 상태 |
| 집 상태 | 귀가/외출/알 수 없음 상태 |
| 위치 상태 | 집근처/외부/알 수 없음 상태 |
| 집 주차 | 집 Wi-Fi 또는 집 GPS 반경 기준으로 집으로 판단된 주차 |
| 외부 주차 | 집 주차 조건에 해당하지 않는 주차 |
| Ongoing Notification | 사용자가 직접 제거하기 어렵고 앱 상태와 연동되어 유지되는 상시 알림 |
| 수동 저장 | 사용자가 자동 감지 없이 직접 현재 주차 위치를 저장하는 기능 |
| 현재 주차 | 현재 유효한 주차 위치로 지정된 주차 이력 |

## 23. MVP 완료 기준

MVP는 다음 조건을 만족하면 완료로 본다.

1. 첫 실행 권한 안내 및 차량 BT 설정 가능
2. 주차 위치 / 주차 이력 / 설정 탭 구성 완료
3. 차량 BT 연결/해제 기반 운행/주차 상태 변경 가능
4. 안정화 지연 설정 적용 가능
5. 집 Wi-Fi, 집 위치, 집 반경, 집 주차장 층수 설정 가능
6. 주차 감지 시 입력 요청 알림 표시
7. 주차 위치 입력 화면에서 집/외부, 지하/지상/기타, 층, 메모, 사진 저장 가능
8. 주차 이력 저장 및 최신순 표시 가능
9. 검색/필터/상세/수정/삭제 가능
10. 외부 주차 Ongoing Notification 동작
11. 외출 시 주차 위치 일회성 진동 알림 동작
12. GPS 좌표 저장 및 외부 지도 앱 열기 가능
13. 데이터 관리 기능 동작
14. SQLite DB 구조 반영
15. 상태 로그 최근 1,000개 보관
